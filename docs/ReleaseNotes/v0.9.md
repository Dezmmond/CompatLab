# CompatLab ArtifactDoctor 0.9

Release date: 2026-06-28

## Summary

CompatLab ArtifactDoctor 0.9 adds static, self-contained HTML reports for
`scan` and `compare`.

This release keeps JSON as the stable machine-readable report format and adds a
human-readable artifact for CI logs, release reviews, and bug reports.

## Added

- `--html PATH` for `scan`.
- `--html PATH` for `compare`.
- Static HTML renderer with embedded CSS and no external scripts, fonts, images,
  CDN links, server, database, or daemon.
- HTML report sections for:
  - report header and command context;
  - diagnostic summary;
  - normalized diagnostics with stable issue codes;
  - bundle dependency resolution;
  - legacy compatibility problems and warnings;
  - compact ELF, target profile, and schema metadata.
- HTML escaping for artifact paths, target/profile fields, dependency names,
  diagnostics, warnings, problems, and other report text.
- CLI error handling for invalid or unwritable HTML output paths.
- Tests for renderer escaping, diagnostic rendering, dependency graph rendering,
  legacy issue rendering, `scan --html`, `compare --html`, combined JSON/HTML
  output, and `--fail-on` behavior with HTML output.

## Current Behavior

Generate a scan report:

```bash
uv run compatlab scan ./dist/my-app \
  --bundle-root ./dist \
  --recursive \
  --html report.html
```

Generate compare JSON and HTML together:

```bash
uv run compatlab compare ./dist/my-app \
  --target ubuntu-2204 \
  --bundle-root ./dist \
  --recursive \
  --fail-on warning \
  --json report.json \
  --html report.html
```

When diagnostics fail the configured `--fail-on` gate, CompatLab writes the HTML
report before exiting with the configured non-zero status.

## Not Included Yet

- Wheel scanning.
- RPM scanning.
- DEB scanning.
- SBOM generation.
- Vulnerability scanning.
- Automatic patching through `patchelf`.
- Artifact execution inside containers.
- Web interface, database, daemon, or server mode.
- Report conversion from existing JSON files.
