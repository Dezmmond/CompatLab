# CompatLab ArtifactDoctor 0.3

Release date: 2026-06-28

## Summary

CompatLab ArtifactDoctor 0.3 adds the first real compatibility comparison
engine for ELF artifacts.

This release turns `compatlab compare PATH --target TARGET` from a target-profile
attachment stub into a practical diagnosis step: it scans the artifact with the
existing `readelf` backend, loads the selected target profile, applies
compatibility rules, reports explainable problems and warnings, writes them to
JSON, and returns CI-friendly exit codes.

## Added

- Real compatibility comparison engine for `compatlab compare PATH --target TARGET`.
- Numeric version comparison for symbol versions:
  - `2.38` is compared as `(2, 38)`;
  - `3.4.29` is compared as `(3, 4, 29)`.
- Maximum required symbol-version selection for:
  - `GLIBC_*`;
  - `GLIBCXX_*`;
  - `CXXABI_*`.
- Architecture normalization for common `readelf` machine names:
  - `Advanced Micro Devices X86-64` -> `x86_64`;
  - `Intel 80386` -> `x86`;
  - `AArch64` and `ARM aarch64` -> `aarch64`.
- Compatibility problems for:
  - architecture mismatch: `wrong.architecture`;
  - dynamic ELF without interpreter: `missing.interpreter`;
  - interpreter not listed by the target profile:
    `profile.interpreter_not_provided`;
  - too-new glibc requirement: `glibc.too_new`;
  - too-new libstdc++/GLIBCXX requirement: `glibcxx.too_new`;
  - too-new CXXABI requirement: `cxxabi.too_new`;
  - direct `DT_NEEDED` library not listed by the target profile:
    `profile.library_not_provided`.
- Warnings for suspicious `RPATH` and `RUNPATH` entries:
  - `bad.rpath.absolute`;
  - `bad.rpath.build_path`;
  - `bad.runpath.absolute`;
  - `bad.runpath.build_path`.
- JSON output for `compare --json` now includes real comparison problems.
- Exit-code behavior for `compatlab compare`:
  - `0` when no `HIGH` or `CRITICAL` compatibility problems are found;
  - `1` when a `HIGH` or `CRITICAL` compatibility problem is found;
  - `2` when the profile is invalid/missing or the artifact cannot be scanned as
    ELF.
- Unit tests for version parsing, architecture normalization, maximum required
  version selection, and the first compatibility rules.

## Changed

- `compatlab compare PATH --target TARGET` now runs the existing ELF scanner
  before comparison instead of returning a stub report.
- Target profile library lists now include `libtinfo.so.6`, which is a common
  direct dependency of `/bin/bash`.
- Scan behavior remains unchanged for `compatlab scan PATH`; scan warnings are
  still reported without turning scan into a compatibility decision.

## Current Behavior

`compatlab compare /bin/bash --target ubuntu-2404` scans `/bin/bash`, checks it
against the Ubuntu 24.04 profile, and reports `Compatibility: PASS` on a modern
x86_64 host where the required symbol versions are within the profile limits.

`compatlab compare /bin/bash --target ubuntu-1804` can report
`glibc.too_new` on modern hosts when the binary requires newer glibc symbols than
Ubuntu 18.04 provides.

`compatlab compare /bin/bash --target ubuntu-1804 --json /tmp/compare.json`
writes the same real comparison problems to JSON, suitable for inspection with
`jq .problems /tmp/compare.json`.

## Not Included Yet

- Recursive dependency resolution.
- Local filesystem library resolution.
- Rootfs or container scanning.
- Wheel, RPM, or DEB scanning.
- HTML reports.
- Web interface, database, daemon, or server mode.
- `pyelftools` backend.
- Go helper implementation.
- Automatic patching through `patchelf`.
