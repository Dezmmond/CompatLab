# CompatLab ArtifactDoctor 1.2

Release date: 2026-07-07

## Summary

CompatLab ArtifactDoctor 1.2 adds RPM package scanning and compatibility
comparison.

The release allows `scan` and `compare` to accept `.rpm` files, inspect package
metadata, discover native ELF entries inside the RPM payload, and reuse the
existing ELF diagnostics/reporting pipeline for every native file.

## Added

- RPM artifact detection.
- RPM metadata parsing for name, epoch, version, release, architecture, summary,
  license, vendor, group, build time, and source RPM where available.
- RPM payload inspection for gzip/plain `newc` cpio payloads.
- Safe payload path validation.
- Native ELF entry discovery inside RPM payloads.
- Safe extraction of native entries for static scanning.
- `scan ./package.rpm` support.
- `compare ./package.rpm --target ...` support.
- RPM metadata and native entry details in JSON reports.
- RPM metadata and native entry sections in HTML reports.
- RPM-specific diagnostic codes.
- CLI, unit, JSON, and HTML tests for RPM workflows.

## Changed

- `scan` and `compare` now dispatch RPM input artifacts by kind while preserving
  existing ELF behavior.
- Diagnostic summary and `--fail-on` behavior now apply to RPM-level reports.
- Native libraries inside an RPM are treated as package-provided libraries for
  direct per-entry comparison.

## Not Included Yet

- DEB scanning.
- SBOM generation.
- Vulnerability scanning.
- Package installation into Docker.
- Running RPM scripts.
- Running RPM payload binaries.
- Automatic `patchelf` fixes.
- RPM signing or repository metadata workflows.
- YUM/DNF dependency solving.
- Public plugin system.
- Breaking JSON schema changes.
- Breaking CLI changes.
