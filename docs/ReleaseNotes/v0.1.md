# CompatLab ArtifactDoctor 0.1

Release date: 2026-06-28

## Summary

CompatLab ArtifactDoctor 0.1 is the first Python-first project skeleton for a
CLI-oriented Linux artifact compatibility checker.

This release establishes the package layout, CLI surface, report/profile models,
initial target profiles, test setup, and developer commands. It intentionally
does not implement the full ELF compatibility engine yet.

## Added

- Python package skeleton with src-style project structure.
- Typer-based CLI entry point:
  - `compatlab scan PATH`
  - `compatlab compare PATH --target TARGET`
  - `compatlab profiles list`
  - `compatlab profiles show TARGET`
- Rich-based terminal output for scan, compare, and profile listing.
- Pydantic models for artifact reports, ELF scan facts, target profiles, and
  compatibility problems.
- JSON report writing for `scan` and `compare` via `--json`.
- Built-in YAML target profiles:
  - `ubuntu-1804`
  - `ubuntu-2004`
  - `ubuntu-2204`
  - `ubuntu-2404`
  - `rocky-9`
  - `astra-17`
  - `sberlinux-9`
- Basic problem taxonomy for future compatibility diagnostics.
- Pytest smoke tests for CLI behavior.
- Ruff configuration for linting and formatting.
- Makefile targets:
  - `make test`
  - `make lint`
  - `make format`
  - `make run-scan`
  - `make run-profiles`
- Packaging configuration so `uv run compatlab ...` works through the declared
  console script.

## Current Behavior

The CLI returns structured stub reports. `scan` records basic file metadata and
prints a compatibility `PASS` placeholder. `compare` loads a target profile and
attaches it to the report, but does not yet perform real ELF compatibility
checks.

## Not Included Yet

- Real `readelf` backend.
- Parsing of ELF headers, interpreter, `DT_NEEDED`, `RPATH`, `RUNPATH`, or
  required symbol versions.
- Compatibility comparison for glibc, libstdc++, CXXABI, architecture,
  interpreter, or libraries.
- Wheel, RPM, DEB, container image, or rootfs support.
- Web interface, database, daemon, or server mode.
