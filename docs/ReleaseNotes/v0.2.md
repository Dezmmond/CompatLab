# CompatLab ArtifactDoctor 0.2

Release date: 2026-06-28

## Summary

CompatLab ArtifactDoctor 0.2 replaces the scan placeholder with the first real
ELF scanner backend.

This release focuses on one practical workflow: scanning a single ELF binary or
shared library with the system `readelf` utility and returning structured facts
through the existing CLI, pretty output, and JSON report model.

## Added

- Real `readelf`-based ELF scan backend for `compatlab scan PATH`.
- Safe command runner around `subprocess.run`:
  - no `shell=True`;
  - timeout support;
  - captured stdout, stderr, and return code;
  - stable `LC_ALL=C`/`LANG=C` environment for parser-safe command output;
  - graceful handling when `readelf` is missing or a command fails.
- Parsers for:
  - `readelf -h`;
  - `readelf -l`;
  - `readelf -d`;
  - `readelf --version-info`.
- ELF metadata extraction:
  - ELF class;
  - endianness;
  - OS ABI;
  - machine/architecture;
  - ELF type;
  - entry point address;
  - dynamic/static signal;
  - program interpreter from `PT_INTERP`;
  - `DT_NEEDED` libraries;
  - `RPATH`;
  - `RUNPATH`;
  - required `GLIBC_*`, `GLIBCXX_*`, and `CXXABI_*` symbol versions.
- Partial-report behavior with warnings when individual `readelf` commands fail.
- Fixture-based parser tests under `compatlab/tests/fixtures/readelf/`.
- CLI smoke test for scanning `/bin/bash`, guarded for hosts where `/bin/bash` or
  `readelf` is unavailable.
- Test coverage tooling:
  - `pytest-cov` dev dependency;
  - terminal coverage report with total percentage;
  - `coverage.xml` output;
  - optional HTML coverage report.
- New Makefile targets:
  - `make coverage`
  - `make coverage-html`
  - `make format-check`
  - `make check`
- `.gitignore` entries for Python cache and coverage artifacts.
- README and README.ru updates for the v0.2 scanner and test workflow.

## Changed

- `compatlab scan PATH` no longer prints `Compatibility: PASS`.
- Scan output now reports:
  - `Scan: OK`;
  - `Problems: N`;
  - `Warnings: N`.
- Compatibility status remains reserved for `compatlab compare`, where a target
  profile is available.
- Pretty terminal output now renders the extracted ELF facts directly.
- JSON output for `scan --json` now includes real ELF metadata.

## Current Behavior

`compatlab scan /bin/bash` extracts real metadata from the host binary, including
ELF header fields, interpreter, needed libraries, RPATH/RUNPATH, and required
symbol versions.

`compatlab compare PATH --target TARGET` still attaches the selected target
profile to the report, but does not yet implement platform compatibility rules.

## Not Included Yet

- glibc, libstdc++, or CXXABI compatibility comparison against target profiles.
- Recursive dependency resolution.
- Local filesystem library resolution.
- Wheel, RPM, DEB, rootfs, or container scanning.
- HTML reports, web UI, database, daemon, server mode, or Go implementation.
- `pyelftools` backend.
- Automatic patching through `patchelf`.
