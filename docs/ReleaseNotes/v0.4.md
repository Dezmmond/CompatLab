# CompatLab ArtifactDoctor 0.4

Release date: 2026-06-28

## Summary

CompatLab ArtifactDoctor 0.4 adds automatic target profile generation from the
current Linux system.

This release teaches CompatLab to detect raw system facts, convert them into a
validated target profile, and use that generated YAML profile in compatibility
checks. The main workflow is now practical on a target or target-like Linux
system without manually writing every profile field.

## Added

- Raw `SystemFacts` model for current-system detection.
- `/etc/os-release` parser for OS identity and version facts.
- Current architecture detection through Python platform facts.
- Best-effort glibc version detection from `ldd --version`.
- `ldconfig -p` parser for provided shared libraries and library paths.
- Dynamic linker detection for common Linux loader paths.
- Available symbol-version extraction from system libraries:
  - `GLIBC_*` from `libc.so.6`;
  - `GLIBCXX_*` from `libstdc++.so.6`;
  - `CXXABI_*` from `libstdc++.so.6`.
- Target profile generation from detected system facts.
- Optional metadata block in generated YAML profiles:
  - generator name;
  - generation timestamp;
  - source type;
  - source OS ID and version;
  - detection backend.
- `compatlab profiles detect` command.
- `compatlab profiles detect --json PATH` for raw `SystemFacts` export.
- `compatlab profiles generate --from-current --name NAME --output PATH`.
- `compatlab profiles validate PROFILE.yaml`.
- `compatlab profiles validate PROFILE.yaml --json PATH`.
- `compatlab compare PATH --target-file PROFILE.yaml`.
- Fixture and CLI tests for profile parsing, generation, validation, JSON
  export, and external profile comparison.

## Changed

- `compatlab compare` now requires exactly one target selector:
  - `--target TARGET` for built-in profiles;
  - `--target-file PROFILE.yaml` for an explicit YAML profile.
- Profile loading now reports missing files, malformed YAML, and model
  validation failures through a shared validation path.
- The existing `readelf` command runner was generalized for small local command
  probes while preserving the `readelf` scanner behavior.
- Built-in profiles remain compatible with the profile model even when metadata
  is absent.

## Current Behavior

On a Linux target or target-like host, users can generate a local target profile:

```bash
uv run compatlab profiles generate --from-current --name local --output /tmp/local.yaml
```

The generated profile can be validated:

```bash
uv run compatlab profiles validate /tmp/local.yaml
```

It can then be used directly for comparison:

```bash
uv run compatlab compare /bin/bash --target-file /tmp/local.yaml
```

Raw detection facts can also be exported for debugging:

```bash
uv run compatlab profiles detect --json /tmp/system-facts.json
```

## Not Included Yet

- Docker image profile generation.
- Rootfs scanning.
- Recursive dependency resolution.
- Local filesystem dependency resolution for scanned artifacts.
- Runtime profile presets with installed packages.
- Wheel, RPM, or DEB scanning.
- HTML reports.
- Web interface, database, daemon, or server mode.
- Automatic patching through `patchelf`.
