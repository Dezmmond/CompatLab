# CompatLab ArtifactDoctor 0.8

Release date: 2026-06-28

## Summary

CompatLab ArtifactDoctor 0.8 adds actionable diagnostics and CI quality gates.

This release keeps the v0.7 bundle dependency graph and adds a normalized
diagnostic layer with stable issue codes, summary counters, and configurable
exit-code behavior for CI pipelines.

## Added

- Stable diagnostic models:
  - `DiagnosticIssue`;
  - `DiagnosticSummary`;
  - `DiagnosticSeverity`;
  - `DiagnosticCategory`.
- JSON report fields:
  - `summary`;
  - `diagnostics`.
- Stable diagnostic issue codes, including:
  - `CL_ARCH_MISMATCH`;
  - `CL_INTERP_MISSING`;
  - `CL_LIB_MISSING`;
  - `CL_SYMBOL_GLIBC_TOO_NEW`;
  - `CL_SYMBOL_GLIBCXX_TOO_NEW`;
  - `CL_SYMBOL_CXXABI_TOO_NEW`;
  - `CL_BUNDLE_AMBIGUOUS_LIB`;
  - `CL_BUNDLE_MAX_DEPTH_REACHED`;
  - `CL_BUNDLE_MAX_FILES_REACHED`;
  - `CL_RPATH_ABSOLUTE`;
  - `CL_RPATH_ESCAPES_BUNDLE`;
  - `CL_RPATH_UNRESOLVED_TOKEN`;
  - `CL_ELF_SCAN_FAILED`.
- CI gate flag for `scan` and `compare`:
  - `--fail-on error`;
  - `--fail-on warning`;
  - `--fail-on never`.
- Diagnostic summary calculation:
  - status;
  - error count;
  - warning count;
  - info count;
  - issue-code counters.
- Dependency-chain attribution for bundle missing and ambiguous dependency
  diagnostics when the graph can be traversed.
- Rich terminal diagnostics and summary sections.
- Bundle runtime path diagnostics for `$ORIGIN` paths that escape the bundle and
  unsupported runtime path tokens.
- Unit and CLI tests for diagnostic models, summary calculation, quality gates,
  JSON output, and fail-on behavior.

## Changed

- `scan` and `compare` now enrich reports with normalized diagnostics while
  preserving existing `problems`, `warnings`, and `dependency_graph` fields.
- `compare` exit behavior is now driven by diagnostics and `--fail-on`; the
  default `error` mode preserves the previous compatibility-failure behavior.
- Warning-only compatibility results can now fail CI with `--fail-on warning`.
- Diagnostic codes are kept visible in terminal output.

## Current Behavior

Fail CI on errors only:

```bash
uv run compatlab compare ./dist/my-app \
  --target-file /tmp/ubuntu-2204-cpp-runtime.yaml \
  --bundle-root ./dist \
  --recursive \
  --fail-on error
```

Fail CI on warnings or errors and write JSON:

```bash
uv run compatlab compare ./dist/my-app \
  --target ubuntu-2204 \
  --bundle-root ./dist \
  --recursive \
  --fail-on warning \
  --json report.json
```

Produce a report without failing because of diagnostics:

```bash
uv run compatlab scan ./dist/my-app \
  --bundle-root ./dist \
  --recursive \
  --fail-on never \
  --json scan-report.json
```

## Not Included Yet

- Wheel scanning.
- RPM scanning.
- DEB scanning.
- SBOM generation.
- Vulnerability scanning.
- Automatic patching through `patchelf`.
- Artifact execution inside containers.
- Docker image mutation, commit, save, or push.
- Arbitrary package installation.
- Full dynamic linker emulation.
- HTML reports.
- Web interface, database, daemon, or server mode.
- Go helper implementation.
