# CompatLab ArtifactDoctor 0.10

Release date: 2026-07-07

## Summary

CompatLab ArtifactDoctor 0.10 is an internal architecture refactoring release.

This release keeps the existing user-facing behavior for ELF scanning,
compatibility comparison, bundle dependency resolution, diagnostics, JSON
reports, HTML reports, and profile generation. The main goal is to make the
codebase easier to understand and extend before adding future artifact types
such as Python wheels, RPM packages, or DEB packages.

## Changed

- Replaced the previous `compatlab/src/...` package layout with a direct
  `compatlab/...` package layout.
- Replaced the Typer CLI dependency with an `argparse`-based command parser.
- Split CLI orchestration into application service modules:
  - `compatlab.services.artifacts` for `scan` and `compare` workflows;
  - `compatlab.services.profiles` for profile listing, detection, generation,
    runtime presets, and validation;
  - `compatlab.services.reports` for diagnostics enrichment, report writing,
    HTML context creation, and bundle dependency problem conversion;
  - `compatlab.services.exceptions` for command exit handling.
- Moved shared Pydantic data models into one central module:
  - `compatlab.models`.
- Replaced dynamic package export discovery with explicit `__init__.py`
  imports and `__all__` definitions.
- Reorganized `compatlab.profile` around workflow responsibilities:
  - `catalog.py`: load built-in and external YAML profiles;
  - `builder.py`: build `TargetProfile` objects from detected `SystemFacts`;
  - `local.py`: detect system facts from the current host;
  - `docker.py`: inspect Docker images, export rootfs archives, and detect
    facts from Docker image rootfs data;
  - `parsers.py`: parse `os-release`, `ldd`, `ldconfig`, and dynamic linker
    information;
  - `rootfs.py`: read and inspect Linux rootfs tar archives;
  - `runtimes.py`: manage runtime package presets for container probing.
- Added a profile package README that documents the new module layout.
- Updated tests to use the new package structure and import paths.
- Updated project packaging configuration so the wheel includes the direct
  `compatlab` package.

## Current Behavior

Existing CLI behavior is preserved.

The main workflows remain:

```bash
compatlab scan ./app
compatlab scan ./dist/my-app --bundle-root ./dist --recursive
compatlab compare ./app --target ubuntu-1804
compatlab compare ./app --target-file ./local.yaml
compatlab compare ./dist/my-app --target-file ./local.yaml --bundle-root ./dist --recursive
compatlab profiles list
compatlab profiles show ubuntu-1804
compatlab profiles detect
compatlab profiles generate --from-current --name local --output local.yaml
compatlab profiles generate --from-image ubuntu:22.04 --name ubuntu-2204-docker --output ubuntu-2204.yaml
compatlab profiles runtime-presets list
compatlab profiles validate ubuntu-2204.yaml
```

`scan` still extracts ELF facts from a local artifact. `compare` still checks a
local ELF artifact against a target Linux profile. `--bundle-root` remains an
optional extension for resolving shared libraries shipped alongside the
artifact. Docker image support remains focused on generating target profiles
from image rootfs data; CompatLab still does not execute the checked artifact
inside Docker.

## Not Included Yet

- Wheel scanning.
- RPM scanning.
- DEB scanning.
- SBOM generation.
- Vulnerability scanning.
- Automatic `patchelf` fixes.
- Running artifacts inside containers.
- Docker image mutation, commit, save, or push.
- Web interface, database, daemon, or server mode.
- Public plugin system.
- Breaking JSON schema changes.
- Breaking CLI changes.

## Next Step

The next development step is to use the stabilized internal structure to add
new artifact scanners incrementally. The intended path is to introduce support
for wheel, RPM, or DEB artifacts without spreading package-specific logic across
the CLI, ELF scanner, diagnostics, report writers, and profile modules.
