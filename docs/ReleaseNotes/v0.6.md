# CompatLab ArtifactDoctor 0.6

Release date: 2026-06-28

## Summary

CompatLab ArtifactDoctor 0.6 adds Docker runtime profile presets.

This release can generate a target profile from a temporary Docker container
after installing a predefined runtime preset, such as common C/C++ runtime
libraries or Python 3 runtime libraries. The source Docker image is not mutated,
committed, or saved.

## Added

- Built-in runtime preset registry:
  - `cpp-runtime`;
  - `python-runtime`.
- Runtime preset inspection commands:
  - `compatlab profiles runtime-presets list`;
  - `compatlab profiles runtime-presets show PRESET`.
- Package manager detection for Docker rootfs exports:
  - `apt-get`;
  - `dnf`;
  - `yum`.
- Install script generation for supported package managers.
- Docker runtime rootfs export flow:
  - create a temporary container;
  - run the runtime preset install script;
  - export the resulting rootfs;
  - cleanup with `docker rm -f`.
- Runtime-aware profile generation:
  - `compatlab profiles generate --from-image IMAGE --runtime-preset PRESET`.
- Runtime-aware raw facts export:
  - `compatlab profiles detect --from-image IMAGE --runtime-preset PRESET --json PATH`.
- Runtime metadata in generated profiles:
  - runtime preset name;
  - installed runtime packages;
  - detected package manager;
  - Docker runtime rootfs export backend.
- Unit tests for preset registry, install script generation, package manager
  detection, Docker runtime orchestration, CLI behavior, and generated metadata.

## Changed

- Docker image profile generation keeps the v0.5 behavior when
  `--runtime-preset` is not provided.
- Generated Docker runtime profiles use `source: docker-runtime-image` and
  `detection_backend: docker-runtime-rootfs-export`.
- `--runtime-preset` is accepted only together with `--from-image`.

## Current Behavior

Users can list available runtime presets:

```bash
uv run compatlab profiles runtime-presets list
```

They can inspect one preset:

```bash
uv run compatlab profiles runtime-presets show cpp-runtime
```

They can generate a runtime-aware target profile:

```bash
uv run compatlab profiles generate \
  --from-image ubuntu:22.04 \
  --runtime-preset cpp-runtime \
  --name ubuntu-2204-cpp-runtime \
  --output /tmp/ubuntu-2204-cpp-runtime.yaml
```

The generated profile validates with the existing validator:

```bash
uv run compatlab profiles validate /tmp/ubuntu-2204-cpp-runtime.yaml
```

It can then be used by compatibility comparison:

```bash
uv run compatlab compare ./dist/my-app --target-file /tmp/ubuntu-2204-cpp-runtime.yaml
```

Raw runtime detection facts can be exported for debugging:

```bash
uv run compatlab profiles detect \
  --from-image ubuntu:22.04 \
  --runtime-preset cpp-runtime \
  --json /tmp/runtime-facts.json
```

## Not Included Yet

- Arbitrary user-provided package installation.
- Dockerfile generation.
- Docker image mutation, commit, save, or push.
- Alpine/musl compatibility modeling.
- Recursive dependency resolution.
- Local filesystem dependency resolution for scanned artifacts.
- Wheel, RPM, or DEB package scanning.
- Artifact scanning inside a running container.
- HTML reports.
- Web interface, database, daemon, or server mode.
- Automatic patching through `patchelf`.
- Go helper implementation.
